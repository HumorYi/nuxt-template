# 架构设计

## 缘由

- 由于 nuxt i18n 插件 只支持单文件，不能在文件内 import 多个子模块文件，给开发和维护带来了不便。为了解决这个问题，特设置此架构。

## 思路

- i18n/locales/zh-Hans.js 单语言文件，每个语言一个文件，适配 nuxt i18n 插件引用文件规则
- i18n/locales/zh-Hans    单语言文件夹，每个语言对应一个文件夹，文件夹内包含该语言的所有翻译文件

## 单语言文件夹

- 按功能划分，能划分出来的，少的放到文件，多的放到文件夹，划分不出来的放在 common.js
- 抽离一些通用的模块，例如：loading、message、day 等
- 业务模块统一以 /app/pages 为准，相当于抄一遍
- 公共组件统一以 /app/components 为准，相当于抄一遍

## 实现

- 开发环境启动时，会默认执行 node-script/locales-watch.js 脚本，只要单语言文件夹下的文件更改，自动同步到对应语言文件
- node node-script/locales-sync.js 执行脚本，以默认语言为准，先同步到所有单语言文件夹，再同步到所有单语言文件
