# 多语言架构设计

## 缘由

- 由于 nuxt i18n 插件 只支持单文件，不能在文件内 import 多个子模块文件
- 需对比 默认语言 与 其它语言 变更，判断哪些是已翻译的，要同步替换，哪些是没翻译的，要提取出来给翻译，翻译好了再替换
- 给开发和维护带来了不便，且多人协同时易起冲突。为了解决以上问题，特设置此架构。

## 架构

```text
i18n
    ├── custom                             # 自定义目录
    │   ├── translated                     # 已翻译内容
    │   │   ├── en.js
    │   │   └── zh-Hant.js
    │   ├── wait-translate                 # 待翻译内容
    │   │   ├── en.js
    │   │   └── zh-Hant.js
    │   └── zh                             # 默认语言，基本架构同 app 目录，其它的都是提取分类
    │       ├── common.js                  # 暂无归类的放公共文件
    │       ├── components                 # 同 app/components
    │       │   ├── CustomTest.js
    │       │   └── test
    │       │       └── CustomUse.js
    │       ├── error.js                   # 错误文案
    │       ├── layouts                    # 同 app/layouts
    │       │   └── default.js
    │       ├── loading.js                 # loading 文案
    │       ├── message.js                 # 信息提示
    │       ├── pages                      # 同 app/pages
    │       │   ├── (auth)                 # 登录认证分组
    │       │   │   ├──(permission)        # 权限分组
    │       │   │      └── user.js
    │       │   │   └── concat.js
    │       │   ├── forbidden.js
    │       │   ├── catchAllRoute.js      # 404 页面 对应 [...slug].vue，[] ... 为非法字符，自行更换命名
    │       │   └── login.js
    │       └── site.js                   # 站点 文案
    ├── i18n.config.ts
    └── locales
        ├── en.js
        ├── zh-Hant.js
        └── zh.js
└── node-script                           # 自定义 node 脚本
    ├── locales-sync.js                   # 同步语言，执行 node node-script/locales-sync.js
    └── locales-watch.js                  # 监听 custom/zh 目录所有文件，同步 locales/zh.js
```

## 实现

- node-script/locales-watch.js
  - 只要 custom/zh 目录下的文件更改，自动同步到 locales/zh.js，开发过程无感知

- node-script/locales-sync.js
  - 以 locales/zh.js 为准，同步到其它语言文件，
  - 从 custom/translated 检索对应语言已翻译内容替换
  - 未翻译内容补充到 custom/wait-translate 对应语言
